McKesson operating rules applied

  • Dallas routes are treated as consistent week-to-week.
  • Whitestown routes can alternate between Week A and Week B.
  • Endpoint logic: Base routes end at Base PLC; Relay routes end at Actual PLC.
  • Active relay routes: Allentown, Buffalo, and Philadelphia are flagged as Whitestown-base / Dallas-actual relay flows.
  • Not serviced by McKesson: Columbus 561, McAllen 549, Bayamon 955, and Caguas 954 are flagged and excluded from route logic.

Route Optimization Engine: this does not ship individual plasma centers. It rebuilds route names / route groups from the current McKesson routes, then compares current route cost vs proposed route cost using actual truck road routes where loaded, plus the Excel Rate Table, workbook miles, fuel surcharge, cases, pallets, Base PLC, Actual PLC, Relay flags, and A/B schedule fields.

Optimization rules applied

  • Current baseline comes from the workbook route, endpoint PLC, miles, linehaul, fuel surcharge, and total weekly cost.
  • Recommended PLC is calculated from closest PLC distance unless the route is flagged as not serviced by McKesson.
  • Fuel opportunity uses the workbook average fuel surcharge percentage and the Excel Rate Table pricing basis.
  • Relay validation flags Allentown, Buffalo, and Philly where Base PLC and Actual PLC differ.
Optimization uses fallback route-mile estimates until you click Load Actual Optimization Routes. Narrow filters first to avoid too many routing API calls.
